# 02_chunking — стратегии чанкинга

## Цель
Из размеченных страниц (выход `01_parsing`) сделать чанки, которые:
- содержат осмысленный объём контекста
- не разрезают таблицы и нумерованные списки
- помнят свой `header_breadcrumb`
- имеют разумный размер (200-800 токенов)

## Стратегии

| Стратегия | Описание |
|---|---|
| `fixed` | Скользящее окно по символам/токенам с overlap. Базлайн |
| `semantic` | Граница чанка там, где сильное семантическое отличие соседних предложений |
| `hierarchical` | По заголовкам разделов (1, 1.1, 1.1.1) с привязкой parent → child |
| `table_aware` | Таблицы — отдельный тип чанка, не режутся, рендер в markdown |
| `recursive` | LangChain RecursiveCharacterTextSplitter |

## Метрики

- **% мусорных чанков** (короче N символов, пустые таблицы, дубли)
- **Распределение длин**
- **recall@5/10** на golden_qa с фиксированным embedder'ом
- **% таблиц, разрезанных пополам**
